Siggy checks the signature.

Constant-time HMAC for the symmetric providers; RSA-SHA256 against PayPal's chain-validated certificate — over the raw bytes, never a re-serialized body.

Your app trusts one thing.

Verified traffic is re-signed with a single Gateway signature and forwarded byte-identical. Everything else gets a 401 and a log line.

Your app is never exposed. Only the gate is.

Four providers, four different locks.

Every provider signs differently — different headers, algorithms, encodings, replay rules. HookGuard speaks each dialect so you don't have to.

A security tool you can audit in an afternoon.

Zero dependencies, provably.

$ go list -deps . | grep -E 'stripe|go-github'

(nothing)

The shipped gateway imports only the Go standard library. Nothing to typosquat, nothing to vendor-audit, no supply chain but Go itself.

A published correctness harness.

Each verifier's verdict is differentially tested against the provider's own official library, across valid, tampered and adversarial payloads. Incumbents don't ship this.

A real trust boundary.

Cryptographic half: the Gateway signature binds the verified provider name — not a forgeable "X-Verified: true" header. Network half: your app has no published port.

Small on purpose.
